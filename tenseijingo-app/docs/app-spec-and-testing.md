# 天声人語エディタ 仕様とテスト運用

## 概要

天声人語の清書を想定した、縦書き専用の原稿用紙エディタです。  
フロントエンドは Tauri + Vanilla TypeScript、保存と履歴管理は Rust 側で行います。

この文書は次の 3 点をまとめています。

- アプリ全体の仕様
- 縦書き特有の記法まわりの挙動
- エージェント向けのテスト実行方法

## 画面構成

### 初回セットアップ

- 初回起動時は保存先フォルダの選択画面を表示
- デフォルト保存先をそのまま使うこともできる
- 任意フォルダを選んで開始することもできる

### ファイル管理画面

- 原稿一覧を表示
- 新規作成
- テキスト読込
- 原稿ごとのプレビュー
- 原稿削除
- 保存先パス表示
- 保存先変更

### エディタ画面

- 35 列 × 18 行を基本とする原稿用紙表示
- 先頭数列の字下げ、末尾列の空きマスを反映
- 一覧へ戻る
- 手動保存
- 名前変更
- 書出
- プレビュー
- 版履歴
- 設定

### 上部インライン表示調整

- 文字サイズ
- 文字太さ
- 罫線の実線 / 点線

### 設定モーダル

- 行頭禁則文字
- 行末禁則文字
- 禁則マスの強調色
- 連数字の大きさ補正
- カーソル位置
- 一マスのプレビュー

### プレビュー

- 整形後の本文を右側パネルに表示
- コピー可能

### 版履歴

- Git ベースの履歴一覧
- 過去版本文の表示
- 任意版への復元

## 編集仕様

### 基本動作

- 縦書きで 1 マス 1 文字を基本に配置
- 改行で次の列へ進む
- 2 桁数字は連字として 1 マスに収める
- カーソル移動、選択、IME 入力に対応
- 自動保存あり
- 明示保存時と自動保存時で保存状態表示を更新
- 保存済表示は次のキー入力で即時解除

### 文字サイズ

- 上部インライン UI から段階変更
- 最大側ではマスにかなり近い大きさまで表示
- 連数字は通常文字とは別に設定モーダル内で補正可能

### 文字太さ

- 文書全体の見た目として調整
- `bold` 機能は廃止済み
- Windows でも段階差が見えるよう補助描画あり

### 罫線

- 薄めの罫線を表示
- 実線 / 点線の切り替えに対応

### カーソル

- 上 / 下 / 左 / 右 を設定可能
- 上下は横バー、左右は縦バー
- 端にぴったり接しないよう、ごく小さい余白を持つ
- 設定モーダル内のプレビューでも確認可能

## 縦書き特有の記法

### 句読点

- `、` と `。` は縦書き用字形で表示
- 表示位置はマス右上寄せ

### かぎ括弧

- `「` は左下寄せ
- `」` は右上寄せ
- 必要に応じて縦書き用字形へ置換

### 丸括弧

- `（` `）` は縦書き用字形で表示
- 横書き向きのまま出ないよう回転対象から除外

### 音引き

- `ー` は縦書き用に回転表示

### 連数字

- 2 桁連続数字を 1 マスに収める
- 通常文字より少し大きめにできる
- ただしマスからはみ出さない上限付き

### 禁則処理

- 行頭禁則文字と行末禁則文字を設定可能
- 問題マスは設定色で強調表示

## 保存とバックエンド仕様

### 保存先

- デフォルト保存先はアプリデータ配下の `manuscripts`
- カスタム保存先へ変更可能
- 保存先変更時に、既存原稿を引き継ぐか選択可能
- 引き継がない場合は旧保存先に原稿を残す

### 原稿データ

- 1 原稿 1 JSON ファイル
- タイトル、本文、更新時刻、文字数、カスタムタイトル状態を保持
- タイトル未固定時は本文先頭から自動生成

### 履歴

- 保存先ディレクトリを Git リポジトリとして管理
- 新規作成、保存、復元を履歴として追跡
- 履歴一覧と本文プレビューを UI から参照可能

### Windows アンインストール

- 原稿と設定はインストール先ではなくアプリデータ側に保存
- NSIS の `Delete application data` は UI から除去済み
- 通常のアンインストールでは原稿・設定を消さない前提

## テスト構成

### フロントエンド

Playwright で Tauri API をモックし、主要画面をスクリーンショット比較します。

現状の主な対象:

- 初回セットアップ
- ファイル一覧
- エディタ本体
- プレビュー表示
- 履歴一覧
- 履歴プレビュー
- 設定プレビュー
- 保存先変更確認ダイアログ
- 保存済表示の解除挙動
- 保存先変更時の invoke 引数

### バックエンド

Rust のユニットテストで次を確認します。

- デフォルト保存先
- 保存先変更
- 既存原稿の移行あり / なし
- 旧保存先への原稿残存
- デフォルト保存先への復帰
- 文字数計算とタイトル自動生成
- 名前変更
- 削除
- 一覧順
- Git 履歴
- 過去本文取得
- 任意版への復元

## エージェント向けテスト手順

### 1. 依存関係

```bash
npm install
npx playwright install chromium
```

### 2. バックエンドテスト

```bash
cargo test --manifest-path src-tauri/Cargo.toml
```

### 3. ビジュアルテスト

```bash
npm run test:visual
```

スナップショット更新が必要なとき:

```bash
npm run test:visual:update
```

### 4. ビルド確認

```bash
npm run build
cargo check --manifest-path src-tauri/Cargo.toml
```

## エージェント向け確認観点

見た目まわりは差分が出やすいので、失敗時は次を重点的に見ます。

- 句読点、かぎ括弧、丸括弧、音引きの向きと位置
- 連数字の大きさ
- カーソル位置と余白
- 罫線の実線 / 点線
- IME 候補位置と本文の重なり
- 設定モーダル内プレビューと実表示の整合

バックエンドまわりでは次を重点的に見ます。

- 保存先変更時の移行有無
- 既存原稿の保持
- Git 履歴が壊れていないか
- 復元後の本文とタイトル
- Windows アンインストール時に消してはいけないデータを削除していないか

## 更新時の運用メモ

- UI や縦書き表示を変更したら Playwright のスナップショット更新が必要になることがある
- 保存仕様や履歴仕様を変えたら Rust テストを追加する
- リリース前は `cargo test`, `npm run test:visual`, `npm run build`, `cargo check` を一通り通す
